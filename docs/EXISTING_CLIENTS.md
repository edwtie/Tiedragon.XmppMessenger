# Existing XMPP Chat Clients

This project is not starting in an empty market. XMPP already has many clients across desktop, mobile and web. The opportunity is to choose a clear product focus instead of trying to copy every existing client.

## Strong Existing Clients

| Client | Platforms | Strength |
| --- | --- | --- |
| Conversations | Android | Mature mobile XMPP client with strong modern XMPP support and audio/video calling. |
| Gajim | Windows, Linux, macOS | Mature desktop client with many XMPP features and plugin ecosystem. |
| Dino | Linux, BSD | Modern desktop chat client with a clean interface and audio/video direction. |
| Monal | iOS, macOS | Modern Apple-platform XMPP client with audio/video support. |
| Converse.js | Browser | Browser-based XMPP client and embeddable web chat. |
| Movim | Web / progressive web app | Social/web-style XMPP client with multi-user chat and cross-client compatibility. |
| Kaidan | Desktop/mobile direction | Modern chat app built around XMPP and Kirigami/Qt. |
| Pidgin | Windows, Linux, macOS | Older multi-protocol client with XMPP support. |
| Spark | Desktop | Older XMPP client, often associated with Openfire deployments. |
| Profanity | Terminal | Power-user console client with OMEMO and MUC support. |

## Observations

- Android is well covered by Conversations and related apps.
- iOS/macOS is covered by Monal and related clients.
- Linux has Dino and Gajim.
- Windows has Gajim, Pidgin, Spark and older multi-protocol clients, but there is room for a modern Windows-first XMPP app.
- Browser/web is covered by Converse.js and Movim.
- Audio/video support exists, but is uneven across clients and platforms.
- Real-time text is a strong accessibility feature, but it is not a headline feature in most mainstream XMPP clients.

## Product Gap For Tiedragon XMPP Messenger

A useful direction is not to compete as a generic clone. The project should focus on:

- Windows-first desktop experience
- clear Dutch/English user interface
- simple setup for non-technical users
- real-time text as a first-class feature
- accessible chat and call flow
- XMPP/Jingle/WebRTC audio and video
- optional self-hosted/private server setup
- careful privacy documentation

## Recommended Positioning

Tiedragon XMPP Messenger should be:

> A Windows-first XMPP messenger for private chat, real-time text, and accessible audio/video communication.

That gives the project a clearer identity than simply "an XMPP client".
